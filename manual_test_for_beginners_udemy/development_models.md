### The most known development models
---
#### Sequencial Software Development(Waterfall vs V-model)
- Both are considered as sequencial development models and the fundamental caractheristic is that one phase only beggins when the previous one is completely ended.
- For software testing this means that the testing phase just beggins at the end of the life cycle.

> ***Waterfall***
>> - Consists in 5/6 phases(it depens of the source): Requirement analysis, System Design, Implementation, Testing, Deplyoment and Maintenance.
>> - If anything is incorrect at the early phases of the cycle, it is necessary to adjust and perform all the steps again.
>> - It is suitable for small teams/companies

> ***V-model*** (also known as verification and validation model)
>> - It's a variant of the waterfall model. For each development phase we have a corresponding testing phase.
>> - This structure brings the testing to the beginning of the development process, but it still have the sequence of phases to be followed.
>> - On the left: Requirement analysis, system design, architecture design, module design, coding (descending)
>> - On the right: Unit testing, Integration testing, System testing, Acceptance testing(ascending)
>> - Suitable for stable, small-to-medium-sized projects where requirements are clearly defined and unlikely to change. 

#### Agile Software Development
- This type of life cycle try to integrate the end user throughout the whole proccess of the software development.
- The idea is to build useful softwares since the first deploy.
- Agile is divided in to types: iterative and incremental

> ***Iterative***
>> - The goal is to develop a useful system since the beginning and along the process adding more functions to it.

> ***Incremental*** 
>> - Develop all the parts of the software and only at the end deliver a useful system. 

> ***Introduction to Scrum***
>> - It is a way to apply the principles of Agile Manifesto. One of the most known and used agile method.
>> - It has this structure:
>>> - Product Backlog: It constains all the requirements of our software
>>> - User stories: The document format in which our requirements will be written: "as a user, I want something, in order to achieve..."
>>> - Sprint: A cycle of user stories that will be developed in a certain period of time(1 to 4 weeks)
>>> - Stand up meeting: Doing daily to tell what is done, what still have to be done and if the team are facing any problem;
