# 🚀 Como se Destacar como QA de Verdade

## 🧠 1. Mindset que muda tudo

QA comum:

> “vou testar o que me pediram”

QA forte:

> “como esse sistema pode falhar em produção?”

---

# 🔥 2. O que você precisa saber na prática

## 🔹 API (Seu maior diferencial)

Você precisa saber:

* Ler documentação (Swagger/OpenAPI)
* Testar com Postman/Insomnia
* Validar:

  * Status code
  * Body
  * Contratos
* Criar testes automatizados de API

Nível acima:

* Autenticação (JWT, OAuth)
* Encadeamento de requisições
* Dados dinâmicos

---

## 🔹 Automação (do jeito certo)

Você precisa saber:

* Cypress ou Playwright
* Estrutura de testes
* Reutilização de código
* Assertions de verdade

Nível acima:

* Mocks
* Interceptação de API
* Evitar testes flaky

---

## 🔹 Fundamentos

Dominar:

* Boundary Value Analysis (BVA)
* Partição de equivalência
* Tabela de decisão
* Teste exploratório

Aplicação prática é o que importa.

---

## 🔹 Debugging

Você precisa saber:

* Ler logs
* Entender erros de API
* Isolar problemas

QA forte aponta a causa, não só o bug.

---

## 🔹 Negócio

Perguntas importantes:

* O que acontece se isso falhar?
* Isso impacta dinheiro?
* Isso quebra o fluxo do usuário?

---

* Explicação do sistema
* O que foi testado
* Como rodar

## 💥 Diferencial

* Testes pensando em produção
